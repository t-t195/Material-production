# コンサル級提案書ジェネレーター

このスキルは「スライドHTMLの作成 → ビルド実行 → 検証・修正」までの全工程を
このセッション内で完結させる。HTML/CSS 部品でスライドを組み、
Playwright + pptxgenjs / pdf-lib で PPTX（投影用）と PDF（事前送付用）を出力する。
ルール詳細はリポジトリ直下の CLAUDE.md、依頼の書式は prompts/slide-generation.md を参照。

## 手順

### 1. 要件の確認
不明なら次を確認する: 案件名（slides/ のフォルダ名になる）、スライド構成
（各ページのタイトル・メッセージ行・本文要素）、宛先・機密表記。
参考デザインのスクリーンショットが提供されたら「部品に分解して」再現する。

### 2. 部品の選定（components/ カタログ）
| # | 部品 | 用途 |
|---|------|------|
| 00 | 表紙 | タイトル・宛先・日付 |
| 01 | 構造表 | 縦書き青ヘッダーで業務・課題を構造化 |
| 02 | 全体工程 | シェブロン矢印＋実施範囲ハイライト |
| 03 | 論理フレーム | 課題（赤）→ありたい姿（青地）→効果（✓） |
| 04 | 投資対効果 | 試算条件＋投資回収ブリッジチャート |
| 05 | 価値定義 | 提供価値×メリット×インパクトのタグ付き表 |
| 06 | 詳細整理 | フェーズ別の詳細整理表 |
| 07 | 章扉 | 目次リストの現在章ハイライト |
| 08 | 比較表 | 案の◎○△×比較＋推奨バッジ |
| 09 | FAQ | 想定問答（Q/Aバッジ形式） |

### 3. スライドHTMLの作成
- `slides/<案件名>/NN-<内容>.html` に連番で配置（ファイル名昇順がスライド順）
- 部品HTMLをコピーして文言を差し替えるのが基本。CSS は
  `../../templates/base.css` と `../../templates/components.css` を参照
- 1スライド1メッセージ。カタログに無い部品を作ったら components/ に見本を追加する

### 4. ビルド（4段パイプライン）
```
npm run build -- slides/<案件名>
```
内部処理: Playwright(Chromium)でHTMLを開く → 倍解像度（deviceScaleFactor=2）で
PNG撮影（等倍は投影時に文字がにじむため禁止） → pptxgenjs で16:9 PPTX化 →
pdf-lib で PDF 化。出力は `output/<案件名>/`。

### 5. 検証
- ビルドログの「⚠ はみ出し検知」警告が出たら該当HTMLを修正して再ビルド
- 生成PNGを Read で目視確認し、レイアウト崩れ・文字切れがないことを確認して報告
