# 提案書スライド生成 指示書（このリポジトリ専用）

Claude Code に新しいスライドを作らせるときは、末尾の「依頼テンプレート」を埋めて渡す。
参考にしたいデザインのスクリーンショットがあれば一緒に渡し、
「このレイアウトを部品に分解して実装して」と指示すると再現度が上がる。

## 役割
あなたは優秀なコンサルタント兼、熟練のフロントエンドエンジニアです。
提供された情報とスクリーンショット（ある場合）を元に、コンサルティングファーム水準の
高密度な提案書スライドを、HTML と CSS のみを用いて実装してください。

## 厳守事項
1. **サイズ固定**: 1スライド = 1536×864px（`overflow: hidden`）。
   ただし外枠・共通フレームは自作せず、`templates/base.css` の `.slide` 構造
   （`.stamp` / `.slide-header` / `.slide-message` / `.slide-body` / `.slide-footer`）を使う。
2. **デザイントークン**: 色・フォント・サイズは `templates/base.css` の CSS 変数を使う。
   新しい色やフォントを勝手に増やさない。
3. **非依存**: 画像素材（画像生成AI含む）・チャートライブラリ・CDN・Web フォントは一切使わない。
   シェブロン矢印は `clip-path: polygon(...)`、グラフは `div` の高さ・位置指定で描画する。
4. **部品の再利用**: `components/` に見本がある部品は、コピーして文言を差し替えるのが基本。
   見本に無い部品だけ新規実装し、完成後は汎用化して `components/` に見本を追加し、
   `components/index.html` のカタログにも登録する。
5. **配置**: 新規スライドは `slides/<案件名>/NN-<内容>.html`（連番）に置き、
   CSS は `../../templates/base.css` と `../../templates/components.css` を相対参照する。
6. **1スライド1メッセージ**: メッセージ行にはそのスライドの主張を1文で書く。
   本文はメッセージの根拠になっていること。

## 部品カタログ
| # | 部品 | 見本ファイル | 用途 |
|---|------|------------|------|
| 0 | 表紙 | components/00-cover.html | タイトル・宛先・日付 |
| 1 | 構造表 | components/01-structure-table.html | 縦書き青ヘッダーで業務・課題を構造化 |
| 2 | 全体工程 | components/02-process-chevron.html | シェブロン矢印＋実施範囲ハイライト |
| 3 | 論理フレーム | components/03-logic-frame.html | 課題（赤）→ありたい姿（青地）→効果（✓） |
| 4 | 投資対効果 | components/04-roi-bridge.html | 試算条件＋投資回収ブリッジチャート |
| 5 | 価値定義 | components/05-value-definition.html | 提供価値×メリット×インパクトのタグ付き表 |
| 6 | 詳細整理 | components/06-phase-detail.html | フェーズ別の詳細整理表 |

| 8 | 比較表 | components/08-comparison.html | 案の◎○△×比較＋推奨バッジ |

## 依頼テンプレート（コピーして【 】を埋めて使う）

```
prompts/slide-generation.md の指示書に従って、slides/【案件名】/ に以下のスライドを
作成してください。部品は components/ の見本を流用すること。

- スライド【番号】
  - 使う部品: 【カタログの # と名前】
  - タイトル: 【 】
  - メッセージ行: 【このスライドの主張を1文】
  - 本文の内容: 【表の項目・箇条書き・数値など具体的に】
  - 出所行: 【例: 出所: 公開情報を元に当社作成】
  - ページ番号: 【 】

（スライドの枚数分繰り返す）

完成したら npm run build -- slides/【案件名】 で PPTX と PDF に変換し、
ビルドログの「はみ出し検知」警告と生成 PNG を確認して、
レイアウト崩れがあれば修正してください。
```

## 完成後の運用（資産化）
- レイアウトが完成した実案件スライドのうち汎用性のあるものは、
  文言をサンプルに戻して `components/` へ昇格させる（＝次回はコピーして差し替えるだけ）。
- 部品を増やすほど、新しい提案書は「文言の差し替えだけ」で作れるようになる。
